git으로 관리하고자 하는 폴더를 클릭하여 git bash here로  연다.
git -> 개발자 사이에서 필수 도구


git init : 해당 폴더를 git으로 관리하겠다.


git add . : 해당 폴더에 변경 내용을 stage(임시공간)에 올린다.

임시공간에 올라가는 내용은 파일을 만들거나, 
파일에 내용을 추가, 수정, 삭제하거나
파일 자체를 삭제하는모든 내용을 말한다.


git commit -m "메시지" : 스테이지에 올라온 내용을 버전으로 기록


git show : 최신 커밋 한 개를 보여준다.


git status : 버전으로 관리가 안되고 있는 파일이나 폴더를 추적


git log : 현재까지 커밋 정보를 확인할 수 있다.


commit <해시>(<데코레이션>) : 커밋해시와 그 커밋을 가리키는 브랜치/태그가 표시된다.

git reset --soft HEAD~1: 마지막 커밋 취소( 단, 스테이지에 코드 유지)
git reset --mixed HEAD~1: 커밋 취소 및 스테이지 취소( 단, 파일은 유지)
git reset --hard HEAD~1: 모두 취소( 파일까지 삭제)

git revert 커밋id : 지정한 내용을 되돌리는 새로운 커밋을 만든다(안전한 방법)
